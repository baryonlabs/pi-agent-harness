# repo-audit — a generated harness

A team that audits a codebase from three independent angles, integrates the findings, and verifies them before reporting. Generated by `pi-agent-harness` and included as a runnable reference.

## Team

| Agent | model | tools | role |
|-------|-------|-------|------|
| `architecture-scout` | claude-sonnet-4-5 | read, grep, find, ls | module boundaries, coupling, layering |
| `security-scout` | claude-sonnet-4-5 | read, grep, find, ls, bash | injection, secrets, authz gaps (read-only bash) |
| `quality-scout` | claude-haiku-4-5 | read, grep, find, ls | dead code, duplication, naming, test gaps |
| `audit-synthesizer` | claude-opus-4-7 | read, grep, find, ls | integrate the three reports into one prioritized audit |
| `skeptic` | claude-sonnet-4-5 | read, grep, find, ls | verify each finding against the code; drop the unsupported |

## Pattern

Fan-out/fan-in → chain verify:

```
parallel: [architecture-scout, security-scout, quality-scout]   # independent angles
   → main reads _workspace/<run_id>/01_*.md and drafts
chain: [audit-synthesizer, skeptic, audit-synthesizer]          # integrate → verify → finalize
```

The scouts are read-only (tools whitelist excludes write/edit), so this harness never mutates your repo — it only produces an audit report. Nothing is committed automatically.

## Run

```bash
pi install npm:@baryonlabs/pi-agent-harness
cd examples/repo-audit
pi
```

Then, inside pi:

```
/repo-audit .
```

Or point it at a subdirectory: `/repo-audit src/`.

Output: `AUDIT.md` (final report) plus intermediate artifacts under `_workspace/<run_id>/`.
