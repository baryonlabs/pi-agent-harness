# Examples

Runnable harnesses generated by **pi-agent-harness**, showing the exact shape of what the factory produces (`.pi/agents/`, `.pi/skills/`, `.pi/prompts/`).

| Example | Domain | Pattern | Runs with |
|---------|--------|---------|-----------|
| [`repo-audit/`](repo-audit/) | Codebase audit | Fan-out/fan-in → chain verify | pi + this package only (zero companion deps) |

## How examples work

Each example is a self-contained project directory containing a `.pi/` folder. To try one:

```bash
# 1. Install pi-agent-harness (bundles the subagent tool)
pi install npm:@baryonlabs/pi-agent-harness

# 2. From inside the example directory (or copy its .pi/ into your own repo), start pi
cd examples/repo-audit
pi

# 3. Trigger the orchestration prompt (filename without .md)
/repo-audit .
```

The orchestration prompt drives the bundled `subagent` tool (`agentScope: "both"`), so the project-local agents in `.pi/agents/` are discovered.

## Domain-agnostic by design

`repo-audit` targets code because every user has a repo to run it on. The **same structure** — specialist agents fanned out in parallel, a synthesizer that integrates, a skeptic that verifies — applies to any domain. Swap the agents and you have a research team, a novel-writing team, or a marketing team. That is the point of a *team-architecture factory*: the patterns are constant, the agents are domain-specific.

To generate your own, install the package and say: `build a harness for <your domain>`.
